Parcours de Réservation : de l'Utilisateur à la Base de Données

Étape 1 : Recherche de Disponibilité

Utilisateur (Front-end)

  • Choisit ses dates d'arrivée et de départ.
  • Clique sur le bouton "Rechercher".

API (Back-end)

  • Reçoit la requête GET /api/chambres avec les dates.
  • Demande à MongoDB les chambres déjà réservées.
  • Demande à MongoDB les chambres disponibles.
  • Calcule le prix selon la saison.
  • Renvoie une liste JSON des chambres disponibles.

Opérations MongoDB

  • db.reservations.find({ ... })
  • db.chambres.find({ _id: { $nin: [...] } })
↓
Étape 2 : Sélection et Remplissage des Informations

Utilisateur (Front-end)

  • Choisit une chambre parmi la liste.
  • Remplit le formulaire avec son nom et son email.
  • Clique sur "Confirmer et Payer".

API (Back-end)

  • Reçoit la requête POST /api/create-payment-intent.
  • Communique avec la passerelle de paiement (ex: Stripe) pour préparer la transaction.

Opérations MongoDB

  • Aucune à ce stade. La réservation n'est pas encore créée.
↓
Étape 3 : Paiement et Création de la Réservation

Utilisateur (Front-end)

  • Est redirigé vers la page de paiement sécurisée.
  • Valide son paiement.

API (Back-end)

  • Reçoit la confirmation de paiement.
  • **Action clé :** Crée un nouvel objet "réservation".
  • Enregistre cet objet dans la base de données.

Opérations MongoDB

  • db.reservations.insertOne({ ... })

Plan de Tests Manuels

Ce document décrit la stratégie de tests manuels mise en place pour valider le bon fonctionnement, la sécurité et la robustesse de l'application. Chaque cas de test est conçu pour vérifier une fonctionnalité clé et la compétence REAC associée.

TEST-FT-01 : Affichage dynamique des chambres

Compétences REAC : CP4, CP7

Objectif (Le Pourquoi)

Vérifier que le front-end communique correctement avec l'API back-end pour récupérer et afficher la liste des chambres, démontrant une architecture découplée fonctionnelle.

Procédure (Le Comment)

  1. Lancer l'application avec docker compose up.
  2. Ouvrir un navigateur à l'adresse http://localhost:8080.
  3. Ouvrir l'inspecteur (F12) et aller dans l'onglet "Réseau".
  4. Observer la requête envoyée à /api/chambres.

Résultat Attendu

La requête réseau doit avoir un statut 200 OK. La page doit afficher dynamiquement les cartes des différentes chambres avec leur nom et leur prix.

Statut du Test OK

TEST-SEC-01 : Sécurité du mot de passe

Compétence REAC : CP7

Objectif (Le Pourquoi)

Valider que les mots de passe des utilisateurs ne sont jamais stockés en clair, mais sont systématiquement hachés par le back-end avant d'être enregistrés en base de données.

Procédure (Le Comment)

  1. Créer un utilisateur via l'API (ou un formulaire).
  2. Se connecter à la base de données avec MongoDB Compass.
  3. Naviguer jusqu'à la collection users.
  4. Inspecter le document du nouvel utilisateur créé.

Résultat Attendu

Le champ password ne doit pas contenir le mot de passe en clair (ex: "password123"). Il doit contenir une longue chaîne de caractères hachée (ex: "pbkdf2:sha256:...").

Statut du Test OK

TEST-INFRA-01 : Déploiement de l'environnement

Compétences REAC : CP1, CP8

Objectif (Le Pourquoi)

S'assurer que l'ensemble de l'infrastructure applicative (Nginx, Flask, MongoDB) peut être monté et détruit de manière fiable et reproductible avec une seule commande.

Procédure (Le Comment)

  1. Cloner le dépôt Git sur une machine propre.
  2. Exécuter la commande docker compose up --build -d.
  3. Vérifier que les 3 conteneurs sont bien en cours d'exécution.
  4. Exécuter la commande docker compose down.

Résultat Attendu

La première commande doit démarrer tous les services sans erreur. La seconde commande doit arrêter et supprimer tous les conteneurs proprement.

Statut du Test OK
